## Problem

Security triage fails when context, evidence queries, and final verdict logic are scattered. SATP was built to coordinate autonomous analysis while keeping the final outcome tied to deterministic policy and evidence.

## Constraints

- Azure REST and Log Analytics access needed to work without assuming optional CLI extensions.
- The workflow had to record query artifacts, debug logs, and triage briefs.
- Automation could recommend and structure, but it could not replace evidence-backed decision policy.

## Approach

SATP ingests Sentinel incidents, alerts, and entities, routes analysis to specialist agents, runs verification KQL against Log Analytics, and applies a DAIR loop across multiple correlation cycles. A verdict ladder normalizes true positive, benign positive, false positive, and requires-investigation outcomes.

## Artifact And Proof

The supporting artifact documents the case-state folder, verification query outputs, Markdown triage brief, debug logs, and MCD draft generation path for true-positive outcomes.

## Outcomes And Status

SATP demonstrates how autonomous triage can be made auditable: every run produces artifacts, every verdict has a policy path, and uncertain cases retain a requires-investigation status instead of being forced into confidence theater.
